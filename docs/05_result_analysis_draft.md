# 5단계: 결과 분석 보고서 초안

## 1. 결과 분석 방법

모델의 성능을 확인하기 위해 전체 데이터를 학습용 데이터와 테스트용 데이터로 나누었다. 학습용 데이터는 나이브 베이즈 분류기가 단어별 조건부확률을 학습하는 데 사용하였고, 테스트용 데이터는 학습에 사용하지 않은 문자에 대해 모델이 얼마나 잘 예측하는지 확인하는 데 사용하였다.

모델 평가는 정확도, 분류 리포트, 혼동행렬을 이용하여 진행하였다. 정확도는 전체 테스트 문자 중 예측이 맞은 비율을 의미한다. 혼동행렬은 실제 라벨과 모델의 예측 라벨이 어떻게 다른지 표로 보여준다.

## 2. 혼동행렬 해석

혼동행렬에서 실제 일반 문자를 일반 문자로 맞힌 경우는 정상적인 예측이다. 실제 스미싱 문자를 스미싱 문자로 맞힌 경우도 정상적인 예측이다.

반면 실제 일반 문자를 스미싱 문자로 잘못 예측하면 사용자가 불필요하게 불안감을 느낄 수 있다. 실제 스미싱 문자를 일반 문자로 잘못 예측하면 위험한 문자를 놓칠 수 있으므로 보안 관점에서는 더 큰 문제가 된다.

따라서 스미싱 탐지 프로그램에서는 단순히 전체 정확도만 보는 것이 아니라, 실제 스미싱 문자를 얼마나 잘 잡아내는지도 중요하다.

## 3. 주요 단어 분석

`05_analyze_results.py`에서는 모델이 각 분류에서 중요하게 본 단어들을 확인한다. 이때 출력되는 단어들은 해당 분류에서 등장할 조건부확률이 높은 단어들이다.

예를 들어 스미싱 문자 쪽에서 `계좌`, `인증`, `링크`, `배송`, `오류`, `확인`과 같은 단어들이 높게 나타난다면, 모델은 이러한 단어들이 포함된 문자를 스미싱으로 판단할 가능성이 커진다.

다만 특정 단어가 등장했다고 해서 무조건 스미싱이라는 뜻은 아니다. 나이브 베이즈 분류기는 여러 단어의 확률을 함께 계산하여 최종 판단을 내린다.

## 4. 오분류 사례 분석

오분류 사례는 `results/misclassified_examples.csv` 파일에 저장된다. 이 파일을 보면 실제 라벨과 모델의 예측 라벨이 다른 문장을 확인할 수 있다.

오분류가 발생하는 이유는 다음과 같이 생각할 수 있다.

- 정상 문자에도 `확인`, `인증`, `배송`처럼 스미싱 문자에 자주 나오는 단어가 포함될 수 있다.
- 스미싱 문자가 매우 짧거나 직접적인 위험 단어를 포함하지 않을 수 있다.
- 나이브 베이즈 분류기는 문맥을 깊게 이해하기보다 단어의 등장 빈도를 중심으로 판단한다.
- 데이터셋에 비슷한 표현이 충분히 없으면 새로운 표현을 정확히 분류하기 어렵다.

## 5. 모델의 장점

나이브 베이즈 분류기는 계산이 빠르고 구조가 단순하다. 또한 조건부확률을 이용하여 분류 결과를 설명할 수 있기 때문에, 확률과 통계 과목에서 배운 개념이 실제 인공지능 모델에 어떻게 적용되는지 확인하기 좋다.

특히 문자 메시지처럼 비교적 짧은 텍스트에서는 단어 빈도만으로도 어느 정도 분류가 가능하다는 점을 확인할 수 있다.

## 6. 모델의 한계와 개선 방향

나이브 베이즈 분류기는 단어들이 서로 독립이라고 가정한다. 하지만 실제 문장에서는 단어의 순서와 문맥이 중요하다. 예를 들어 `링크를 누르지 마세요`와 `링크를 눌러 확인하세요`는 모두 `링크`라는 단어를 포함하지만 의미는 다르다.

따라서 더 정확한 모델을 만들기 위해서는 형태소 분석기를 사용하여 한국어 단어를 더 정확히 나누거나, 문맥을 고려할 수 있는 다른 인공지능 모델과 비교하는 방향으로 확장할 수 있다.
