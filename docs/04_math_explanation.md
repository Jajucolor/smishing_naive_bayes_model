# 4단계: 보고서용 수학 원리 설명

## 1. 조건부확률

조건부확률은 어떤 사건이 이미 일어났다는 조건 아래에서 다른 사건이 일어날 확률이다.

예를 들어 문자에 `링크`라는 단어가 포함되어 있을 때, 그 문자가 스미싱 문자일 확률은 다음과 같이 생각할 수 있다.

```text
P(스미싱 문자 | 링크 포함)
```

이는 `링크`라는 조건이 주어졌을 때 문자가 스미싱일 가능성을 의미한다.

## 2. 베이즈 정리

베이즈 정리는 조건부확률을 이용해 원인과 결과의 관계를 뒤집어 계산할 수 있게 해준다.

```text
P(스미싱 | 단어) = P(단어 | 스미싱) × P(스미싱) / P(단어)
```

문자 분류에서는 모든 분류에 대해 같은 입력 단어를 비교하므로, 실제 계산에서는 분모인 `P(단어)`를 생략하고 점수를 비교할 수 있다.

```text
P(스미싱 | 단어) ∝ P(스미싱) × P(단어 | 스미싱)
```

## 3. 나이브 베이즈 분류기

문자 하나에는 여러 단어가 들어 있다.

나이브 베이즈 분류기는 이 단어들이 서로 독립적으로 등장한다고 가정한다. 현실에서는 단어들이 완전히 독립은 아니지만, 이 단순한 가정 덕분에 계산이 쉬워진다.

```text
P(스미싱 | 단어들)
∝ P(스미싱) × P(단어1 | 스미싱) × P(단어2 | 스미싱) × ...
```

마찬가지로 일반 문자에 대해서도 계산한다.

```text
P(일반 | 단어들)
∝ P(일반) × P(단어1 | 일반) × P(단어2 | 일반) × ...
```

마지막으로 두 값을 비교하여 더 큰 쪽으로 분류한다.

## 4. 라플라스 스무딩

문제는 어떤 단어가 특정 분류에서 한 번도 등장하지 않았을 때 발생한다.

예를 들어 `계좌`라는 단어가 일반 문자 데이터에서 한 번도 등장하지 않았다면 다음 확률은 0이 된다.

```text
P(계좌 | 일반) = 0
```

그러면 다른 단어들의 확률이 아무리 높아도 전체 곱셈 결과가 0이 된다.

이를 막기 위해 라플라스 스무딩을 사용한다.

```text
P(단어 | 분류)
= (해당 분류에서 그 단어가 등장한 횟수 + 1)
  / (해당 분류의 전체 단어 수 + 전체 단어 종류 수)
```

즉, 모든 단어가 최소 1번은 등장한 것처럼 보정하여 확률이 0이 되는 문제를 방지한다.
